## 과제 요약
- git 설치 버전확인
- git congig 활용 global로 이름, 이메일 설정해보기
- git config --list로 설정 사항 확인하기
- 폴더 생성후 git 활성화 하고 파일 생성후 add, commit 해보기, git log로 커밋 로그 확인
- head의 현재 위치를 확인하는 방법
- head의 상태 참조로 커밋의 내용의 비교해보고, 과거 커밋과 현재 커밋 비교해보기
- deatached head 만들어 보고 head의 위치 확인해보기
- deatached head 상태에서 브랜치를 만들어 작업을 보존하기
- feture 브랜치 생성, 파일 커밋후 main브랜치로 돌아가 커밋상황을 비교해보기
- feture 브랜치의 파일과, main브랜치의 동일 파일을 소정하고 충돌상황을 만들고 merge해보기
- 원격저장소를 생성후 로컬 디렉토리와 연결해보기
- 원격저장소에 파일을 push 해보고, 원격 자장소에서 로컬로 파일을 pull해보기
- feture브랜치를 원격저장소에 main에 push 해보고, pull request을 요청해보기, pr을 처리하고 원격저장소 main으로 merge해보기


## git 실습 피드백
* commit 메세지나 다른 bash 명령어에 '~'같은 불필요한 캐릭터 쓰지말것
* ls 명령어를 쓸시에 ls 단독 보다는 ls -alh 의 명령어를 사용하여 세부사항까지 출력하여 쓰는것을 추천
  - ls -a (all로 숨김까지 모두 표시) , ls -l (표시 파일의 권한 ,용량,이름등 세부사항 출력),
  - ls -lh(l로 표시된 파일들의 용량들을 kb mb gb등 단위로 표시해줌)


# 프로젝트 구조

1. til - 월요일 강의는 개인 til로 대체 하며, 나머지 일자는 데일리 스크럼을 기준으로 회고를 작성
2. Assignments - 과제에 필요한 내용들을 모아 놓음, 과제 피드백 사항을 기록
3. 기타 파일 - 프로젝트 내에 쓰이는 각종 리소스들, 그리고 나중에 디스코드로 질문할 것들 모음 혹은 디스코드 질문 크롤링
4. 이론 정리 - 월요일 수업시간에 배운 이론을 정리하며, 과제 도중 필요한 이론, 따로 개인공부를 위한 이론을 정리
5. 클래스에서 생성한 파일 - 월요일 수업시간 도중 생성한 파일들의 모음(강사의 파일도 따로 업로드 예정)

# 프로젝트 파일 원칙

1. 텍스트 파일의 확장자는 기본적으로 .md를 유지할것
2. 모든 텍스트 파일의 확장자는 생성 날짜를 네이밍 앞쪽에 표기할것
3. 기타 및 클래스 생성에 필요한 파일들은 제외하고 네이밍은 구체적으로 할것(날짜 표기는 안함)

# TIL 폴더의 원칙

1. 최신화는 반드시 매일 수업종료후 해놓을것
2. 각 날짜의 til은 아침 데일리 스크럼을 기준으로 작성할것
3. 월요일은 강의 날짜로 데일리 스크럼이 없으므로 개인의 강의의 느낀점, 어려움을 작성할것
